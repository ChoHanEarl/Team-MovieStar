2. 기획하기
ㄴ 주제, 기능, 역할분배, 기간설정, 변수명, 함수명, 레이아웃, 아키텍쳐, 클래스명,

Github로 협업하기
1. 팀장이 레포 만들고 Collaborator로 초대
2. 초대받은 팀원은 수락하여 레포 공유
3. 공유받은 이후 clone으로 받아 로컬에서 개발 시작하기
ㄴ 공유된 레포를 써야하므로 팀원은 로컬에 따로 프로젝트를 만들지 않는다.
4. 각자 로컬에서 개발 진행시 branch를 만들어서 개발 진행
5. 기1ch의 내용을 github에 commit, push 진행
push이후 github에서 pull Request 작성
팀장은 pull request를 확인하여 merge진행
(merge는 해당 branch 내용을 main branch에 합치는 과정)
6. 팀장이 merge를 했다면 팀원들은 각자의 로컬에서 pull을 진행하여 통합된 코드로 업데이트
(pull받기 전까지 개발을 하면 충돌이 난다)
7. 본인기 개발한 내용이 merge되고 pull을 완료했다면 새로운 기능 개발을 한다.

주의사항
1. 되도록이면 합치는 작업은 팀장혹은 1명이 관리한다.
2. 동일한 파일을 두 명이상 동시에 작업하지 말것

권한자가 mainTest 올리기 -> 최종본 Main에 올리기
